Data processing

Summary of results

The processing occurred on Matlab using the data recorded using the cellphones. In total, more than 100 datasets have been collected from the indie-pocket app, from 12 different users, amounting to about 400 5-second bins of labelled data.

Using a KNN classifier in the 7-principal components domain, the prediction accuracy was 95.6%, as shown in the confusion matrix below.

The iid table contains a random phone id, the app version and basic information about the smartphone:

The sensor_data table contains all recording rows:

  • The _id variable is the row number.
  • The status_id field corresponds to the activities (tens) and pocket locations (units).
  • The sensorName field describes which sensor data is stored in the row.
  • The phase variable is incremented each time the activity changes or when the recording is paused and resumed.
  • The accuracy field is not used at this time.
  • The sensors field contains the sensors readout formated as a character string.
  • The timestamp field is filled by the OS upon sensor readout and represented in nanoseconds.

Pre-processing

Clean-up and integrity check

Data are first retrieved from the app vie the server. They are extracted into the Matlab workspace as a Table

conn = sqlite(dbfile);

C = fetch(conn, 'SELECT * FROM sensor_data');

T = cell2table(C, 'VariableNames',...
							 {'line' 'label_num' 'phase' 'sensor' 'void' 'data_s' 'timestamp'});

The rows are sorted chronologically and invalid timestamps are removed

T = sortrows(T,'timestamp','ascend');

T = T(T.timestamp ~= 0, :);

For each sensor type i, the data_s string of each row is parsed to retrieve the sensor measurement.

S{i} = extract_data_snsrlog(T, snsr_list(i), n_axes(i));

% Excerpt from extract_data_snsrlog():
format_str = ['[' repmat('%f, ', 1, n_axes-1) '%f]'];

for i = 1:N
	dat(i,:) = sscanf(data_s(i), format_str)';
end

The data are then separated in phases of continuous recording and similar labels.

Chopping

For each phase, the data are chopped into bins of 5 seconds each.

One bin is discarded at the beginning and end of each continuous recording phase to retain only valid information.

Common timebase interpolation

To create a consistent amount of sensor data at each bin, the timeseries of each sensor are interpolated in a consistent time base. For IMU data (accelerometer and gyroscope) the sampling period is set to IMU_DT which is currently 2ms. Other sensors are currently not in use because they are not consistently present on all phones. All IMU data rows are resampled at this frequency in a common timebase using linear interpolation.

When discontinuities of more than 500% are detected in the sampling period, the bin is discarded.

Frequency domain analysis

The IMU data are passed into the frequency domain using Fast Fourier Transform (FFT). The amplitude of the frequency signal is calculated, to discard phase information.

Principal Components Analysis (PCA)

To reduce its dimensionality, the data is projected onto its principal components using PCA. In order to perform this step, all sensor data are reshaped into a single-column vector with a fixed length. This is possible thanks to the the consistent timebase accross all recording rows.

Classifier training

The data is fed to a series of classifiers to compare their performance. Among the candidates were the following classifiers:

  • Discriminant analysis
  • Decision trees
  • Support vector machine (SVM)
  • Nearest neighbor, multiple classes (KNN)
  • Boosted trees and bagged trees

Cross-validation

The training occurs using a five-fold cross-validation, meaning that 80% of the data are randomly picked and used for training, while the remaining 20% are kept for validation. This is repeated five times over. Prediction accuracy is calculated as the percentage of correct classifications.

Results

The best performance is obtained using the KNN classifier, with single neighbor, Euclidean distance metric and equal weighting. The effect of the number of predictors is shown on the parallel coordinates plot, revealing the importance of including at least five PCs. This yields a classification accuracy of 95.6%. Restricting the predictors to only the first three PCs yields a performance of 90.9% instead.
